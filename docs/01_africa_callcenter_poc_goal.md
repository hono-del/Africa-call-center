# 01. アフリカコールセンターPoC ゴール定義

## 1. PoCの目的

アフリカの自動車販売代理店のコールセンター業務に対して、**AIナレッジ検索・回答支援システム**のイメージを伝えること。

実際の検索精度・AI回答の正確さそのものを検証するPoCではなく、「担当者が問い合わせにどう対応し、そのログがどう改善サイクルにつながるか」という**業務フローと画面体験を伝えるためのモック**である。

## 2. 対象ユーザー

| ユーザー | Phase 1での扱い |
|---|---|
| コールセンター担当者 | メインユーザー。問い合わせ対応時に検索・回答支援ツールを使う |
| 管理者 | メインユーザー。問い合わせ傾向・未解決テーマを可視化画面で確認する |
| 販売店スタッフ | 将来フェーズ。Phase 1では対象外 |
| エンドユーザー（顧客） | 将来フェーズ。Phase 1では対象外（自己解決ポータル等は次フェーズ構想） |

## 3. Phase 1の範囲

### やること
1. コールセンター担当者が問い合わせ対応時に使う**検索・回答支援ツール**（`/assist`画面）を提供する
2. 問い合わせ内容・検索キーワード・提示回答・解決可否などの**ログを収集する**
3. 収集ログを分析し、**FAQ改善・マニュアル改善・AI回答精度改善**につなげる導線（`/analysis` `/suggestions`）を提供する

### 対応言語・チャネル
- 言語: 英語・フランス語・日本語（日本語は日本側関係者がPoCを理解するための言語であり、現地オペレーター向けの主要言語ではない）
- チャネル: 音声通話 + テキストチャネル（WhatsApp/SMS等を想定）

### 対象ドメイン
- 自動車販売代理店の問い合わせ対応（車種・部品・整備・保証等）
- 将来的に自動車以外の商材を扱う代理店にも展開できるよう、汎用性を意識した設計とする（ただしPhase 1のモックデータ自体は自動車ドメインで作成する）

## 4. このモックで伝えるべき価値

1. **担当者の負担軽減**: 症状・質問を入力するだけで、AIが関連情報を検索し、回答候補・確認質問・参考資料・次アクションを提示することで、対応の迷いや調べ物の時間を減らせるイメージ
2. **対応品質の底上げ**: 経験の浅い担当者でも、AIの提示する確認質問・次アクションに沿うことで、一定水準の対応ができるようになるイメージ
3. **継続的な改善サイクル**: 個々の対応ログが蓄積されることで、「どこにナレッジギャップがあるか」「どのFAQ・マニュアルを改善すべきか」が自動的に見えてくるイメージ
4. **経営層・管理者への可視性**: 管理画面を通じて、問い合わせ傾向や未解決テーマが一目でわかり、改善の意思決定に使えるイメージ
5. **多言語対応の可能性**: 英語・フランス語という複数言語圏にまたがるアフリカ市場でも、同じ仕組みが機能するイメージ（日本語は社内理解用として提示）

## 5. やらないこと（Phase 1のスコープ外）

- 実際のAI検索エンジン・LLM APIとの本番連携（モックデータ・疑似応答で代替する）
- 実際のCTI/CRM/WhatsApp Business API等、外部システムとの本番接続
- 販売店スタッフ・エンドユーザー向け画面の実装（設計上の余地は残すが、Phase 1では作らない）
- 動画キャプション生成機能（既存モックにあったが今回は除外）
- 対象国別の通貨・法規制（データ保護規制等）への個別対応
- 認証・アクセス制御の本格実装（ログイン画面等はモック上の簡易表示にとどめる）
- 通話音声そのものの録音・音声認識（テキスト化されたログを前提としたモックとする）
- パフォーマンス・スケーラビリティの検証（PoCのため対象データ量は小規模を想定）
